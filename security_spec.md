# 專案安全性規格 (Security Specification)

## 數據不變量 (Invariants)
1.  **個人隱私**：只有使用者本人或管理員可以讀取/修改 `/users/{userId}` 中的資產（點數、信箱）。
2.  **卡池完整性**：普通玩家只能修改卡池中的 `cards`、`pointPrizes` 與 `remainingPacks` 欄位，且必須是「減少」的操作。
3.  **元數據鎖定**：普通玩家不得修改卡池的價格 (`price`)、名稱 (`name`) 或抽卡限制。

## 髒話/攻擊測試案例 (The Dirty Dozen)
1.  攻擊者嘗試讀取別人的 `/users/someone_else` 資料 -> **拒絕**。
2.  攻擊者嘗試將自己的點數修改為 999999 -> **拒絕** (必須經由合法的扣點邏輯)。
3.  攻擊者嘗試將卡池價格修改為 0 點 -> **拒絕**。
4.  攻擊者嘗試在未登入狀態下抽卡 -> **拒絕**。
5.  攻擊者嘗試修改已售罄卡池的狀態 -> **拒絕**。
